##### VRRP（虚拟路由冗余协议）

通过部署多个网关的方式来解决单点故障问题，避免造成业务的中断，提高网络可靠性。
既可以实现网关的备份，又能解决多个网关之间互相冲突的问题。

VRRP协议报文：**Advertisement报文，其目的IP地址是224.0.0.18。**

VRRP备份组，由一个Master设备和多个Backup设备组成，master设备负责数据的转发；
backup设备做冗余，当主设备发生故障时，会切换成master设备，承载数据的转发。
通过**优先级Priority来决定主备关系，优先级越大越优先（默认值为100）**

Master设备会周期性（1s）向224.0.0.18发送Advertisement报文；
若超过Master_Down_Interval失效时间（大约3s），备设备无法收到主设备发送的报文，会切换成主设备。
如果原Master故障恢复，则会发生主备回切（原来的主设备抢占）

------

默认情况下，当Master设备上行链路故障时，VRRP无法主动感知，不会发生主备切换，造成业务中断。
解决方案
**联动功能：利用VRRP的联动功能监视上行接口或链路故障，主动进行主备切换，避免业务中断。**

------

VRRP负载分担
通过配置不同的备份组，使在某个备份组当中的备设备成为新备份组的Master，这样就可以分担网络中流量。
与主备备份方式的不同点在于：
负载分担方式需要建立多个VRRP备份组，各备份组的Master设备分担在不同设备上；
单台设备可以加入多个备份组，在不同的备份组中扮演不同的角色。

------

VRRP配置注意
接口的物理IP和虚拟IP（网关地址）不能冲突！
**在同一个VRRP备份组当中的虚拟IP必须相同！！！**
**根据优先级决定主备关系**
**可配置抢占延迟和联动功能（追踪接口）**

**VGMP**

 VGMP 用于双活 / 主备，VGMP 会将多个 VRRP 备份组绑定为**VGMP 管理组**，实现**整设备主备切换**（而非单个 VRRP 组独立切换），适用于双机场景：

双活、双主（负载均衡）/主备备份

##### GRE（通用路由封装）隧道技术 

在原来报文的基础上，插入预设的头部信息
VPN（虚拟专用网络）：通过逻辑上的方式解决物理专线的成本和可靠性问题

GRE的应用场景：
**解决异种网络的传输问题（IPv4和IPv6）**
扩展了受跳数限制的路由协议的工作范围，支持企业灵活设计网络拓扑

GRE本身只能靠关键字字段（key）来做隧道两端的身份认证
自己不支持加解密技术，但是可以结合IPSec实现数据的加解密。



#### 关键命令：

##### 一、基础主备 VRRP 备份组配置（单备份组，一主多备）

1. **接口下创建 VRRP 备份组并配置虚拟 IP（核心）**

  ```
  interface GigabitEthernet 0/0/0  // 进入网关接口（如三层交换机VLANIF/路由器物理口）
   ip address 192.168.1.2 255.255.255.0  // 配置接口物理IP（与虚拟IP同网段，不能冲突）
   vrrp vrid 1 virtual-ip 192.168.1.1  // 创建VRRP备份组1，配置虚拟IP（网关地址），同组虚拟IP必须一致
   vrrp vrid 1 priority 120  // 配置优先级（默认100，值越大越优先，主设备配更高优先级）
   vrrp vrid 1 preempt-mode timer delay 5  // 配置抢占延迟（5s，避免故障恢复后频繁主备回切）
   quit
  ```

  

2. **备设备基础配置（仅需配置同组虚拟 IP，优先级默认 100 即可）**

  ```
  interface GigabitEthernet 0/0/0
   ip address 192.168.1.3 255.255.255.0
   vrrp vrid 1 virtual-ip 192.168.1.1  // 必须与主设备同VRID、同虚拟IP
   quit
  ```

  **要点**：`vrid` 为备份组编号，同组设备`vrid`必须一致；物理 IP 与虚拟 IP 不可冲突。

##### 二、VRRP 联动追踪（监视上行接口 / 链路，解决上行故障不切换问题）

针对文档中**“Master 上行链路故障，VRRP 无法主动感知”** 问题，配置**接口追踪**，上行接口故障时自动降低 Master 优先级，触发主备切换。

```
interface GigabitEthernet 0/0/0
 ip address 192.168.1.2 255.255.255.0
 vrrp vrid 1 virtual-ip 192.168.1.1
 vrrp vrid 1 priority 120
 vrrp vrid 1 track interface GigabitEthernet 0/0/1 reduced 30  // 追踪上行接口0/0/1，该接口故障则优先级降低30
 vrrp vrid 1 preempt-mode timer delay 5
 quit
```

**参数说明**：`reduced 30` 表示被追踪接口故障时，本设备 VRRP 优先级降低 30；若主设备优先级从 120→90，低于备设备默认 100，触发备设备抢占成新主。

##### 三、VRRP 负载分担配置（多备份组，设备交叉为主 / 备）

文档中负载分担核心是**多备份组 + 不同设备为各备份组 Master**，以下以**两台设备、两个备份组**为例，实现流量分担：

**设备 A（备份组 1 主，备份组 2 备）**

```
interface GigabitEthernet 0/0/0
 ip address 192.168.1.2 255.255.255.0
 // 备份组1：主设备，优先级120
 vrrp vrid 1 virtual-ip 192.168.1.1
 vrrp vrid 1 priority 120
 // 备份组2：备设备，优先级默认100
 vrrp vrid 2 virtual-ip 192.168.1.254
 quit
```

**设备 B（备份组 1 备，备份组 2 主）**

```
interface GigabitEthernet 0/0/0
 ip address 192.168.1.3 255.255.255.0
 // 备份组1：备设备，优先级默认100
 vrrp vrid 1 virtual-ip 192.168.1.1
 // 备份组2：主设备，优先级120
 vrrp vrid 2 virtual-ip 192.168.1.254
 vrrp vrid 2 priority 120
 quit
```

**业务侧配置**：终端设备分别配置网关为`192.168.1.1`（备份组 1）和`192.168.1.254`（备份组 2），实现流量分担。

##### 四、VRRP 关键查看与调试命令

##### **查看 VRRP 备份组状态（核心）**

```
display vrrp  // 查看所有VRRP备份组的主备状态、优先级、虚拟IP等
display vrrp interface GigabitEthernet 0/0/0  // 查看指定接口的VRRP信息
display vrrp brief  // 简洁查看VRRP主备状态（推荐日常排查）
```



##### 五、VGMP 联动 VRRP 配置（双机双活 / 主备，增强可靠性）

##### **VGMP 会将多个 VRRP 备份组绑定为VGMP 管理组，实现整设备主备切换（而非单个 VRRP 组独立切换），适用于双机场景：**

1. **启用 VGMP 并创建管理组**

```
vgmp enable  // 全局启用VGMP
interface GigabitEthernet 0/0/0
 vrrp vrid 1 virtual-ip 192.168.1.1
 vrrp vrid 1 priority 120
 vrrp vrid 1 bind vgmp group 1  // 将VRRP组1绑定到VGMP管理组1
 vrrp vrid 2 virtual-ip 192.168.1.254
 vrrp vrid 2 bind vgmp group 1  // 将VRRP组2绑定到同一VGMP管理组
 quit
```

2. **配置 VGMP 抢占与延迟**

```
vgmp group 1 preempt-mode timer delay 3  // VGMP管理组抢占延迟3s
```

#### GRE 隧道基础配置（结合 VRRP 实现跨网冗余）

1. ##### **全局启用 GRE**

   ```
   gre enable  // 华为设备全局启用GRE协议
   ```

2. ##### **创建 GRE 隧道接口**

   ```
   interface Tunnel 0/0/1
    ip address 10.0.0.1 255.255.255.0  // 隧道接口逻辑IP（两端同网段）
    tunnel-protocol gre  // 指定隧道协议为GRE
    source GigabitEthernet 0/0/2  // 隧道源接口（本地公网/互联口）
    destination 202.103.100.2  // 隧道目的IP（对端设备互联IP）
    gre key simple 123456  // GRE身份认证关键字（两端必须一致，文档提及GRE仅靠key认证）
    quit
   ```

3. ##### **隧道与 VRRP 联动**

   ##### **将 GRE 隧道接口作为 VRRP 的被追踪接口，实现隧道故障时 VRRP 主备切换：**

```
interface GigabitEthernet 0/0/0 vrrp vrid 1 track interface Tunnel 0/0/1 reduced 40 quit
```
